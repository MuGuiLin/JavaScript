JS执行环境 与 作用域链

			什么是执行环境：
			
				执行环境（execution context）是javascript中最为重要的一个概念。
				执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为。
				每一个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。
				虽然我们的代码无法访问这个对象，但是解析器在处理数据时会在后台执行它。

				全局执行环境是【最外围的一个执行环境】。根据ECMScript实现所在的宿主环境不同，
				表示执行环境的对象也不一样。
			
				每一个函数都有自己的执行环境。当执行流进一个函数时，函数的环境就会被推入一个环境栈中。
				而在函数执行之后，栈将其环境弹出，把控制权返还给之前的执行环境。
				
				当代码在一个环境中执行时，会创建变量对象的一个【作用域链（scope chain）】。
			
			什么是作用域链：
			
				作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。
				全局变量总是最后一次被搜索的！因为它是一层一层的向上找！所以全局变量的执行效率不高，尽量少用！
			 	环境变理(最里面的变量和函数)，可以一层一层的(有顺序了)向上追溯(访问查找)，可以访问它的上一级环境(变量和函数)名或平级环境(变量和函数)
			 	而上一级 环境变几量 就不能访问下一级的环境变量！
			
			
			
			
			var color1 = 'red';//执行环境，color1 window对象（基本顶层的执行环境）
		
			function changeColor() //每个函数都有一个执行环境（variable obj）；
			{
				var color2 = 'green';
				
				function swapColor() //在这里又产生了一个执行环境（variable obj）；
				{
					var color3 = color2;
					color2 = color1
					color1 = color3;
					//color3作用域是：swapColor函数之下 【三级作用域】
					//在这里color1、color2、color3都可以访问！
					
				}
				//这里可以访问color1 和color2 但不能访问color3
				alert(swapColor()); //它的作用域是：changeColor函数之下 【二级作用域】
			}
			//这里只能访问color1;是整体的执行环境 
			swapColor(); //它的作用域是：window 【一级作用域】
			
			//通过以上得出结论：【作用域链】
			//注：全局变量总是最后一次被搜索的！因为它是一层一层的向上找！所以全局变量的执行效率不高，尽量少用！
			//	环境变理(最里面的变量和函数)，可以一层一层的(有顺序了)向上追溯(访问查找)，可以访问它的上一级环境(变量和函数)名或平级环境(变量和函数)
			//	而上一级 环境变理 就不能访问下一级的环境变理！
